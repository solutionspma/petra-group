The Document Depot reads documents.json from the repo. To show the green **Install profile** button
for someone before you change JSON and redeploy, use **Netlify environment variables** (free tier).

After you push the site with the `netlify/functions/profile-flags` function deployed:

--- Using a .env file (repo template) ---

1. In the **petra-group** folder, copy the template:
      cp .env.example .env

2. Edit **.env** (gitignored): uncomment and set either `EMAIL_PROFILE_IOS_LIVE_JSON` or one or more
   `EMAIL_IOS_LIVE_*` lines. Truthy values: 1, true, yes, live.

3. Import into Netlify (either way):

   • **CLI** (from petra-group, after `netlify login` and `netlify link`):
        netlify env:import .env

     To overwrite keys that already exist on the site:
        netlify env:import .env --replace-existing

   • **Dashboard:** Site configuration → Environment variables → look for **Import** / “Import from .env”
     (wording varies) and upload or paste the same file.

4. Redeploy is usually **not** required for function env; Kevin should refresh the depot page.

--- Option 1: Netlify CLI (from your machine) ---

Prereqs: install the CLI once (`npm i -g netlify-cli`) or use `npx netlify`. From the **petra-group** repo root:

  netlify login              # once per machine
  netlify link               # once: pick this site (creates .netlify/state.json; gitignored)

Set Kevin’s iOS row to “live” in **production** (default `env:set` applies to all scopes, including functions):

  netlify env:set EMAIL_IOS_LIVE_KEVIN_DICKERSON 1 --context production

Or one JSON blob (zsh/bash; escape inner double-quotes):

  netlify env:set EMAIL_PROFILE_IOS_LIVE_JSON "{\"kevin-dickerson\":true}" --context production

Useful:

  netlify env:list
  netlify env:unset EMAIL_IOS_LIVE_KEVIN_DICKERSON --context production

CI / scripts: create a personal access token in Netlify (User settings → Applications → Personal access tokens),
export `NETLIFY_AUTH_TOKEN`, then run the same `netlify env:set` commands (no browser login).

--- Option 2: Netlify UI ---

1. Netlify → your site → **Site configuration** → **Environment variables**

2. Choose ONE approach:

   A) Single JSON (easiest for several people at once)
      Name:  EMAIL_PROFILE_IOS_LIVE_JSON
      Value: {"kevin-dickerson":true}
      (Add more ids: {"kevin-dickerson":true,"troy-dixon":true})

   B) One variable per leader (suffix = leader id with UNDERSCORES instead of hyphens)
      Name:  EMAIL_IOS_LIVE_KEVIN_DICKERSON
      Value: 1
      (Same as leader id "kevin-dickerson" in documents.json)

   Truthy values: 1, true, yes, live

3. Save (or after CLI `env:set`). Functions usually pick up new env values on the next request (no full site rebuild required).

4. The actual .mobileconfig file must still exist at the path in documents.json (e.g.
   /assets/documents/email-profiles/kevin-dickerson.mobileconfig) or the button will appear but the link 404s.

5. When you later set status to "live" in documents.json and deploy, you can remove the env override.

Local / non-Netlify hosts: this endpoint is not available; use documents.json status only or Supabase later.
