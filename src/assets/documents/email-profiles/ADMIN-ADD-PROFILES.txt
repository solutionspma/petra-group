DOCUMENT DEPOT — EMAIL PROFILES (ADMIN)

1. Obtain the configuration profile
   - GoDaddy Email / Microsoft 365: use the provider’s “mobile” or “iPhone” setup flow and download the .mobileconfig file (or export from your admin console if applicable).

2. Add the file to the site
   - Place the file under: src/assets/documents/email-profiles/
   - Name must match documents.json path, e.g. troy-dixon.mobileconfig, harry-christophe.mobileconfig, deirdre-carter.mobileconfig
   - Rename GoDaddy’s long download name to that exact filename before committing.

3. Update data
   - Open src/data/documents.json
   - Under emailProfiles → people → find the leader
   - Set the matching download entry "status" to "live"
   - Ensure "path" matches the file path from the site root (e.g. /assets/documents/email-profiles/troy-dixon.mobileconfig)

4. Build and deploy
   - Run: npm run build
   - Deploy the dist folder (e.g. push to Netlify).
   - Commit BOTH the updated documents.json AND the new .mobileconfig binaries (git add). If the file is missing on the host, the Install link will fail — it is not enough to change JSON alone.

5. iPhone behavior
   - Users open the Document Depot on Safari, tap the profile link, allow the download, then Settings → General → VPN & Device Management (or Profile) → Install.
   - Netlify should serve .mobileconfig with Content-Type application/x-apple-aspen-config (configured in netlify.toml).

6. Android (Samsung, etc.)
   - There is no equivalent one-tap profile for consumer email in most cases.
   - Use the server-settings template and/or provider documentation; Outlook and Samsung Email support Exchange and IMAP using the published hostnames.

7. Android guided setup (same mail settings file as source data)
   - When the hosted .mobileconfig is plain XML plist, the site reads it in the browser and builds copy/paste server steps.
   - GoDaddy (and many providers) ship a **signed** profile (PKCS#7 binary). Browsers cannot unpack that, so server lines will not auto-load unless you add a fallback in documents.json:
   - On the same object as "downloads" for that leader, add "mailSettingsForAndroid": { "protocol": "imap", "emailAddress": "user@domain.com", "incoming": { "host": "mail.example.com", "port": 993, "ssl": true, "username": "user@domain.com" }, "outgoing": { "host": "mail.example.com", "port": 465, "ssl": true, "username": "user@domain.com" } }
   - Link: /portal/android-email-setup.html?leader=<leaderId> — also reachable as **Guided setup** next to each Android row in the depot.

8. Android step guide (phone / tablet / laptop browser)
   - Public page: /portal/email-setup-cards.html?leader=<leaderId> (e.g. kevin-dickerson)
   - Each step is one full card (no flip). Server/host/port lines from the staff mail file are merged in when that file is readable.
   - Custom narrative: add "androidFlipTutorial" JSON next to each person in documents.json, or upload JSON via master Email profile admin.
   - Template: flip-tutorial-TEMPLATE.json in this folder. Placeholders: {{fullName}} {{firstName}} {{email}} {{company}}

Do not commit real passwords or secrets into the repository. Profiles may contain server hostnames but should not embed plaintext passwords in publicly hosted files unless your security policy allows it.
